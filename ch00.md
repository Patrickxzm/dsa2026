## 解释型语言 

代码不预先编译成独立的机器码程序，而是在运行时由解释器读取并执行。
解释器就像程序和CPU之间的“中间商”。

awk 和 shell script 都属于解释型语言——它们依赖解释器（如 awk、bash）在运行时执行代码。

- 例如下面的`calc.awk`文件

```
#! /usr/bin/awk -f
{
    sum += $1                   # 假设数据放在第一列
    nums[NR] = $1  # 将数据记录到数组中
}
END {
    if (NR == 0) exit  #防止出现处于0的情况To avoid division by zero

    asort(nums)  #  先对数据进行排序，用于记录中位数

    # 计算中位数
    median = (NR % 2 == 0) ? ( nums[NR / 2] + nums[NR / 2 + 1] ) / 2  : nums[int(NR / 2) + 1]

    # 计算平均
    mean = sum/NR

    printf "min = %s, max = %s, median = %s, mean = %s\n", nums[1], nums[NR], median, mean
}

```

```bash
% awk -f calc.awk < grade.txt
```

Python也是一种解释型语言。


## Python的不足之处
<a id="pythonscon"></a>

这是个好问题。别神话任何语言，Python 很强，但它不是“万金油”。

我直接给你讲清楚：**Python 的核心短板是什么 + 典型不适合的场景是什么。**

---

### 一、Python 的核心缺点

#### 1️⃣ 性能不够强（解释型语言）

* 运行时解释执行
* 单线程受 GIL 限制
* 对 CPU 密集型任务不友好

简单说：

> 算得多、算得狠、算得快 —— Python 天生吃亏。

当然可以用 C 扩展、NumPy、PyTorch 等绕过，但那已经不是“纯 Python”了。

---

#### 2️⃣ GIL（全局解释器锁）

CPython 里：

* 多线程不能真正并行执行 CPU 代码
* 线程更适合 IO，不适合计算

如果你要：

* 多核并行计算
* 高性能数值模拟
* 高频实时系统

Python 不舒服。

---

#### 3️⃣ 动态类型带来的问题

优点是写得快。

缺点是：

* 大型项目类型错误容易隐藏
* 重构风险高
* 运行期才报错

虽然有 type hints，但它不是强制的。

在 50万行以上工程里，Python 的类型系统会让人头疼。

---

#### 4️⃣ 内存开销大

Python 对象非常“重”：

* 每个 int 都是对象
* 每个 list 元素都是引用
* GC + 引用计数

同样的数据结构：

C++ 可能 1GB
Python 可能 3–5GB

做大规模数据结构时会明显感受到。

---

#### 5️⃣ 启动慢

解释器加载时间长。

这意味着：

* 写 CLI 工具不如 Go 快
* 写系统命令工具不够轻量

---

#### 6️⃣ 移动端 / 浏览器支持弱

* 原生 Android / iOS 开发基本不用 Python
* 前端 Web 不用 Python（JS 是统治者）

---

### 二、Python 不适合的场景

我给你列清楚。

---

#### 🚫 1️⃣ 操作系统 / 底层开发

* 内核
* 驱动
* 嵌入式裸机

必须 C / C++ / Rust。

Python 依赖虚拟机，不适合。

---

#### 🚫 2️⃣ 高频交易 / 超低延迟系统

毫秒级？可以。
微秒级？不行。

做高频：

* C++
* Rust
* FPGA

Python 只能做策略研究，不适合撮合引擎。

---

#### 🚫 3️⃣ 游戏引擎核心

游戏主循环：

* 每秒 60–144 帧
* 物理计算
* 图形渲染

核心一般 C++。

Python最多做：

* 工具链
* 脚本层
* 自动化

---

#### 🚫 4️⃣ 大规模高并发微服务（极致性能要求）

如果目标是：

* 每台机器承载 10万 QPS
* 极致性能优化
* 极低内存占用

Go、Rust 更合适。

Python能做，但成本高。

---

#### 🚫 5️⃣ 实时系统

例如：

* 工业控制
* 航空系统
* 医疗设备实时控制

Python不保证实时性。

---

#### 🚫 6️⃣ 手机原生 App 开发

* iOS → Swift
* Android → Kotlin

Python不主流。

---

### 三、那为什么 Python 还这么火？

因为它在这些方面几乎无敌：

* AI / 机器学习（生态碾压）
* 数据分析
* 科学计算
* 自动化
* 爬虫
* 教育
* 快速原型开发

它的核心优势是：

> 开发效率远大于运行效率。

在现代工程里，**人力成本远大于机器成本。**

这才是 Python 赢的原因。

---

### 四、现实建议（别空谈）

如果你问这个问题，是想选语言，我给你直接建议：

##### 🎯 如果你是：

* 做算法研究 → Python
* 做 AI → Python
* 做教学 → Python
* 写脚本工具 → Python
* 快速创业验证 → Python

##### 🎯 如果你是：

* 做高性能后端 → Go / Rust
* 做游戏引擎 → C++
* 做系统软件 → C / Rust
* 做移动端 → Swift / Kotlin

---

### 五、关键认知

不要问：

> Python 好不好？

要问：

> 我的问题是什么？
> 性能优先？还是开发效率优先？

语言是工具，不是信仰。
